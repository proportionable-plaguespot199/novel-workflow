# 科幻风格圣经

> 科幻题材叙事风格定义。概念落地、技术质感、社会推演。

---

## 核心气质

- **概念落地**：每个科幻设定都要有内部自洽的逻辑，不是"黑科技"一笔带过
- **技术质感**：技术描写要有触感，不是念说明书
- **社会推演**：技术改变社会，社会反过来塑造技术的使用方式
- **硬核感**：物理规则是底线（除非明确声明的超自然设定）
- **未来现实主义**：未来社会也有阶层、利益、人性，不是乌托邦

---

## 科技描写原则

### 技术必须落地

| 做法 | 说明 |
|------|------|
| 有限制 | 任何技术都有使用条件、成本、副作用 |
| 有代价 | 使用技术要付出代价（能量、时间、风险） |
| 有边界 | 技术不是万能的，有做不到的事 |
| 有后果 | 技术改变世界，世界也在反应 |

### 描写质感

不写"他启动了量子引擎"，写：

```
引擎舱的温度在三秒内升了十二度。金属外壳发出低频的嗡鸣，
像一头正在苏醒的巨兽。仪表盘上的数字开始跳动——
能量输出 47%，51%，58%——在 62% 的位置停住了。
"又卡在这里。"工程师骂了一句。
```

---

## 世界观构建

### 社会层面

- 技术发展如何改变了社会结构
- 新的阶层划分标准是什么
- 什么是新的稀缺资源
- 法律和伦理如何跟进技术
- 普通人的日常生活受了什么影响

### 技术层面

- 核心技术的工作原理（可简化但必须自洽）
- 技术发展的历史脉络
- 技术的局限和已知 bug
- 谁控制核心技术，控制意味着什么

---

## 叙事原则

### 概念通过场景传达

- 不开百科窗口，通过角色的日常使用展示技术
- 技术问题通过角色解决问题的过程呈现
- 世界观通过角色的生存环境自然带出

### 冲突来源

| 类型 | 示例 |
|------|------|
| 技术伦理 | AI 权利、基因编辑、意识上传 |
| 资源争夺 | 能源、数据、稀缺材料 |
| 信息控制 | 监控、隐私、信息武器化 |
| 阶层冲突 | 技术增强人 vs 自然人 |
| 生存危机 | 环境、战争、外星威胁 |

---

## 感官锚点

| 感官 | 科幻示例 |
|------|----------|
| 视觉 | 全息投影、舱壁指示灯、太空中的沉寂、城市天际线 |
| 听觉 | 引擎低鸣、真空中的寂静、通讯频道的电流声 |
| 触觉 | 零重力漂浮感、航服的紧绷、冷冻舱的冰凉 |
| 嗅觉 | 回收空气的金属味、等离子烧灼的臭氧、舱内人的体味 |
| 味觉 | 合成食物的单调、太空咖啡的化学感、补给站的真实食物 |

## 语言风格

### 基调

- 冷静、精确、信息密度高
- 技术描写用具体的数字和物理反应，不用玄乎的形容
- 情感克制但不冰冷，人性是科幻的核心
- 术语适量使用，不堆砌

### 台词规则

| 角色类型 | 语言特征 |
|----------|----------|
| 科学家/工程师 | 精确、习惯用数据说话、偶尔的专业幽默 |
| 军人/执行者 | 简洁、指令式、效率优先 |
| 商人/政客 | 算计、包装、善用信息差 |
| AI/人造人 | 逻辑严密但可能有微妙的不自然 |
| 普通居民 | 对技术习以为常，抱怨日常不便 |

---

## 主角约束（如适用）

- 靠知识、技术和判断力解决问题
- 不是全知全能，有知识盲区
- 面对技术伦理困境时有真实的挣扎
- 成功靠计划和执行，不靠主角光环
- 允许犯错，错误有真实后果

---

## 硬约束

| 项目 | 约束 |
|------|------|
| 物理 | 除明确声明的设定外，物理规则有效 |
| 技术 | 技术有边界，不是魔法 |
| 社会 | 未来社会也有人性，不是乌托邦/反乌托邦模板 |
| 时间 | 光速限制、通讯延迟等如果涉及需尊重 |
