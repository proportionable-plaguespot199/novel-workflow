# 全题材通用规则

> 所有题材共享的基础创作规则。题材特定的规则在各自的 style_bible.md 和 forbidden_patterns.md 中定义。

---

## 移动端适配

| 规则 | 说明 |
|------|------|
| 段落长度 | 3-5 行一段，超过 5 行必须拆分 |
| 句子节奏 | 长短交替，禁止连续 3 个以上等长句 |
| 对话呼吸 | 对话与动作/叙述穿插，不连续超过 5 轮纯对话 |
| 视觉留白 | 场景切换处空一行，章节内重大转折处空一行 |

---

## 段落呼吸

### 禁止密不透风

- 不连续超过 3 段纯叙述（无对话、无动作描写）
- 不连续超过 5 段纯对话（无叙述、无动作穿插）
- 每个场景至少有 1 次环境/感官描写打断叙述流

### 节奏模板

```
[动作/事件] -> [反应/对话] -> [环境/感官] -> [推进]
```

不必严格遵循，但要意识到节奏变化。单一节奏持续太久读者会疲劳。

---

## 信息源优先级

始终按以下顺序判定信息可信度：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 用户本轮的明确要求 | 最高优先级，覆盖一切 |
| 2 | 用户贴出的正文/设定补丁 | 当轮有效 |
| 3 | status_card.md | 当前状态的权威来源 |
| 4 | 最近相关正文章节 | 因果链的直接依据 |
| 5 | 题材设定文件（style_bible 等） | 风格和规则约束 |
| 6 | 本地资料库/知识库 | 辅助参考 |
| 7 | 联网查证结果 | 只补硬事实 |
| 8 | 通用经验与类比 | 最低优先级 |

### 冲突处理

- 最新正文 > 旧设定/旧大纲
- 用户明确要求重构时，以新要求为准，但标注覆盖了哪些旧设定
- 联网结果只补真实世界事实，不覆盖已写的剧情事实

## 全局禁止模式

### 叙事禁区

| 禁止 | 说明 |
|------|------|
| 机械降神 | 解决问题必须在前文有迹可循，不能临时发明设定 |
| 系统/外挂保姆化 | 辅助工具不是全自动代打器 |
| 原著复读机 | 如有原著参考，不得照搬桥段 |
| 课件式开头 | 不用大段背景介绍开场，用场景/动作/对话切入 |
| AI 说教结尾 | 不以总结、感悟、升华、哲理收尾 |

### 语言禁区

| 禁止 | 说明 |
|------|------|
| 疲劳词堆砌 | 高识别词单章同一词最多 1 次 |
| 同义反复 | 不连续用不同词说同一件事 |
| 模板化狠话 | 不用通用口号，台词必须贴身份 |
| 连续同句式 | 不连续 3 句以上用相同句式开头 |
| 空洞修饰 | "非常""极其""无比"等不提供信息的修饰词尽量删除 |

### 角色禁区

| 禁止 | 说明 |
|------|------|
| 配角工具人 | 配角必须有自己的算盘、恐惧、筹码 |
| 全员降智 | 不为推剧情让角色集体犯蠢 |
| 无脑种马 | 不写无逻辑的感情线 |
| 反派排队送死 | 反派必须有脑子，有试探、误判、反扑 |

### 数据禁区

| 禁止 | 说明 |
|------|------|
| 模糊收益 | 不写"暴涨""海量""难以估量" |
| 数据通胀 | 数值增长必须有逻辑依据 |
| 战力崩坏 | 战力表现必须与已建立体系一致 |
| 无记录变更 | 状态变化必须同步到相关状态文件 |

---

## 质量底线

### 每章必须包含

- 至少 1 项推进：信息、地位、资源、伤亡、仇恨、关系
- 至少 3 种感官描写
- 至少 1 个具体的利益变化或代价
- 明确的下章钩子或悬念

### 三章内必须提供

- 明确的反馈兑现（打脸、收益、反转、地位变化均可）
- 至少 1 个伏笔推进或回收
- 可感知的主角状态变化

---

## 文件维护规则

| 触发条件 | 必须更新的文件 |
|----------|---------------|
| 正文完成 | status_card.md |
| 角色状态变化 | characters.md |
| 伏笔新增/回收 | hooks_pool.md |
| 资源/数值变化 | ledger.md |
| 时间推进 | timeline.md |
| 章节交接 | handoff.md |

不能只更新正文而忽略状态文件。
