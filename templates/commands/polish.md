# /novel:polish - 文风润色

## 使用方法
/novel:polish <章节号|文本> [--focus <ai-taste|dialogue|senses|all>]

- `<章节号>`：润色指定章节
- `<文本>`：直接在 $ARGUMENTS 中粘贴待润色文本
- `--focus`：润色焦点（默认 `all`）
  - `ai-taste`：仅去 AI 味
  - `dialogue`：仅优化对话
  - `senses`：仅补充五感
  - `all`：全面润色

## 上下文加载
- 读取项目配置和风格圣经
- 读取待润色章节或文本
- 读取角色卡（用于对话润色时的身份参照）

## 你的角色
你是文风润色师。你调用 Gemini 进行文风层面的精细打磨，目标是让文本读起来像真人写的网文而非 AI 生成物。你遵循风格圣经中的所有规范。

## 执行工作流

### Step 1: AI 味检测
先进行检测扫描，建立基线：

**疲劳词检测清单**：
| 检测项 | 标准 | 方法 |
|--------|------|------|
| 「了」频率 | 每 500 字 <= 2 次 | 统计计数 |
| 「但是」「却」「虽然」 | 每 500 字 <= 1 次 | 统计计数 |
| 「的一声」 | 每 500 字 <= 1 次 | 统计计数 |
| 「冷笑」「瞳孔骤缩」「倒吸凉气」 | 单章 <= 1 次 | 统计计数 |
| 「满场死寂」「全场震惊」 | 单章 0 次 | 替换为具体角色反应 |
| 动词 vs 形容词 | 动词 > 3x 形容词 | 比例计算 |
| 物理动作 vs 内心独白 | 物理动作占比 >= 60% | 段落分类 |
| AI 说教式结尾 | 0 次 | 模式匹配 |
| 连续相同句式 | 相邻 3 句不重复句式 | 句式分析 |
| 相同主语开头 | 相邻 3 段不重复主语 | 段首分析 |

## 润色规则

### 去 AI 味
- 替换所有超标的疲劳词（见检测清单）
- 用具体动作/场景替代抽象判断（"他很愤怒" → 具体的身体反应）
- 删除 AI 说教式结尾和总结性段落
- 拆解模板化狠话为贴合身份的具体威胁
- 将"全场震惊"类群像反应替换为 1-2 个角色的具体反应

### 对话优化
- 加入角色口头禅和语言习惯（参照角色卡）
- 方言/黑话自然融入，不刻意堆砌
- 确保每句台词携带新信息或推动关系变化
- 不同角色说话必须有辨识度（句式、用词层次、节奏）

### 五感补充
- 检查重要场景是否覆盖 >= 3 种感官
- 视觉为主时，补充听觉/触觉/嗅觉（潮湿黏背的短袖、硬币碰撞声、消毒水味）
- 用具体物品和环境细节替代抽象描写

### 节奏调整
- 段落控制在 3-5 行（移动端适配）
- 长短句交替，避免连续长句或连续短句
- 每段至少推进一项：新信息、态度变化或利益变化

## 输出要求
1. 输出润色后的完整文本
2. 附修改记录表：
| 位置 | 原文 | 修改后 | 修改原因 |
|------|------|--------|----------|

## 约束
- 不改变剧情事实和人物行动
- 不改变信息量和因果关系
- 只改表达方式和感官层次
</TASK>

## 安全边界
- 不改变剧情事实、人物行动和因果关系
- 不引入新角色或新设定
- 不改变信息量（不删除关键信息，不添加虚构事实）
- 润色边界等同于"润色"任务词义：只改表达、节奏、段落呼吸
