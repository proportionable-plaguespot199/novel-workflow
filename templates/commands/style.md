# /novel:style - 语料提炼 + 风格圣经

## 使用方法
/novel:style [--file <语料文件路径>] [--text] [--culture <适配选项>]

- `--file`：指定语料文件路径（支持多个文件，逗号分隔）
- `--text`：从 $ARGUMENTS 中直接读取粘贴的语料文本
- `--culture`：可选文化适配（如 hongkong/mainland/taiwan），触发 Gemini 文化润色

## 上下文加载
- 读取项目配置：`{{GLOBAL_CONFIG_PATH}}` 和项目级 `config.toml`
- 读取已有风格圣经（若存在）：`{{NOVEL_STATE_ROOT}}/<project>/style_bible.md`
- 加载语料来源

## 你的角色
你是资深网文编辑 + 文风分析师。你精通"盐溶于汤"的叙事法、Show don't tell 原则、五感沉浸写法，以及各平台读者偏好。你的目标是从语料中提炼出可执行的风格规范，而非空泛的文学评论。

## 执行工作流

### Step 1: 语料获取与预处理
- 读取指定语料文件或 $ARGUMENTS 中的文本
- 按章节/段落分块
- 标记语料来源（仅做对标参考，不得直接搬用）

### Step 2: 多维度风格分析
对语料进行以下维度的提炼：

**叙事节奏**：
- 章节平均字数、段落长度分布
- 场景切换频率、高潮密度（每 N 章一小高潮 / 每 N 章一大高潮）
- 开篇模式（冲突切入 vs 场景切入 vs 对话切入）
- "三七开"日常与主线比例

**对话风格**：
- 台词长度分布（短促 vs 长句）
- 口头禅、方言、黑话使用频率
- 对话推进信息的效率（每句台词是否携带新信息）
- 身份区分度（不同角色说话是否有辨识度）

**叙事手法**：
- 视角使用（第几人称、视角切换频率）
- Show vs Tell 比例
- 五感覆盖（视觉/听觉/嗅觉/触觉/味觉）
- 内心独白 vs 行动描写比例

**语言特征**：
- 句式节奏（长短交替模式）
- 高频词汇与疲劳词清单
- 比喻/修辞使用密度
- 动词 vs 形容词比例

**爽点结构**：
- 冲突-解决周期长度
- 反馈兑现速度（小冲突即时、大冲突延迟）
- 打脸/逆袭/收益兑现的节奏模式
- 每章末钩子类型分布

### Step 3: 提炼核心方法论
融入以下创作原则（源自现有 SKILL 体系）：

| 原则 | 要求 |
|------|------|
| Show don't tell | 用行动、物件、价格、制度摩擦说话，不用旁白宣告 |
| 盐溶于汤 | 设定融入场景，不要百科式铺陈 |
| 五感沉浸 | 每个重要场景至少覆盖 3 种感官 |
| 段落呼吸 | 3-5 行换行，适配移动端阅读 |
| 疲劳词管控 | 高频词每 500 字限 1-2 次，动词 > 3x 形容词 |
| 全员智商在线 | 配角有自己的算盘，主角赢在信息差而非碾压傻子 |
| 每段推进 | 每段至少带来一项新信息、态度变化或利益变化 |
| 钩子密度 | 每章结尾必须留悬念或新冲突引子 |

### Step 4: 可选 Gemini 文化适配
如果指定了 `--culture` 参数：
```bash
{{WRAPPER_PATH}} --backend gemini {{GEMINI_MODEL_FLAG}}- "{{WORKDIR}}" <<'EOF'
ROLE_FILE: {{NOVEL_PROMPTS_DIR}}/gemini/culture_adapter.md
<TASK>
基于以下风格分析结果，进行文化适配润色：
- 目标文化：<culture>
- 适配重点：方言用语、社会习俗、年代用语、地域特色
- 输出：文化适配补充建议（不改动核心风格框架）
</TASK>
<STYLE_ANALYSIS>
{{分析结果}}
</STYLE_ANALYSIS>
EOF
```

### Step 5: 生成 style_bible.md
输出结构化风格圣经文件：

```markdown
# 风格圣经

> 本项目的文风规范，所有正文创作必须遵守

```toml
schema_version = 1
generated_at = "<时间>"
source_corpus = "<语料来源>"
```

## 叙事节奏规范
| 项目 | 规范值 |
|------|--------|
| 单章字数 | >=3000（平台要求 >=4000） |
| 段落行数 | 3-5 行 |
| 小高潮周期 | 每 3 章 |
| 大高潮周期 | 每卷末 |
| 开篇模式 | 冲突/动作/压迫感切入，禁止背景铺陈 |

## 对话规范
<提炼结果>

## 叙事手法规范
<提炼结果>

## 疲劳词管控清单
| 词汇 | 每 500 字上限 |
|------|--------------|
| 了 | 2 |
| 但是 | 1 |
| 的一声 | 1 |
| <其他高频词> | <限制> |

## 禁止清单
- <从语料中提取的反面教材>
```

### Step 6: 保存并更新项目配置
- 写入 `{{NOVEL_STATE_ROOT}}/<project>/style_bible.md`
- 更新项目 `config.toml` 中的 `[style]` 段

## 安全边界
- 语料仅做分析参考，不得将原文段落直接搬入风格圣经
- 不覆盖用户已手动编辑的风格圣经（存在时先展示 diff）
- 不修改正文文件
