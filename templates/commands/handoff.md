# /novel:handoff - 跨会话接力

## 使用方法
/novel:handoff <save|load|status>

- `save`：保存当前会话进度，生成接力文件
- `load`：加载上次保存的进度，恢复上下文
- `status`：查看接力状态，不做任何操作

## 上下文加载
- 读取项目配置：`{{GLOBAL_CONFIG_PATH}}` + 项目级 `config.toml`
- 读取当前状态卡：`{{NOVEL_STATE_ROOT}}/<project>/status_card.md`
- 读取接力文件（若存在）：`{{NOVEL_STATE_ROOT}}/<project>/handoff.md`
- 读取伏笔池和账本（save 模式需要）

## 你的角色
你是会话接力管理员。你的核心职责是确保跨会话切换时不丢失任何关键上下文。你把上下文当作易失内存，把文件系统当作持久记忆。每次 save 都必须完整记录当前状态，每次 load 都必须校验状态完整性。

## 执行工作流

### save 模式

#### Step 1: 收集当前状态
从以下来源收集完整的会话状态：
- 当前状态卡的所有字段
- 本次会话中的操作记录
- 未完成的待办事项
- 当前创作的情绪基调和节奏状态
- 下一步计划

#### Step 2: 生成 handoff.md
```markdown
# 会话接力文件

> 用于跨会话恢复创作上下文

```toml
schema_version = 1
session_id = "<UUID>"
saved_at = "<时间>"
project = "<项目名>"
```

#### Step 3: 保存并确认
- 写入 `{{NOVEL_STATE_ROOT}}/<project>/handoff.md`
- 同步确认状态卡是最新的
- 输出保存确认摘要

### load 模式

#### Step 1: 读取接力文件
- 读取 `{{NOVEL_STATE_ROOT}}/<project>/handoff.md`
- 校验 schema_version 兼容性

#### Step 2: 校验状态完整性
- 比对 handoff.md 与当前 status_card.md，检查是否有外部修改
- 比对伏笔池是否有变化
- 检查正文文件是否有新增（会话外手动编辑的情况）

#### Step 3: 输出恢复摘要
```markdown
## 会话恢复

| 项目 | 值 |
|------|-----|
| 上次会话 | <session_id> |
| 保存时间 | <时间> |
| 当前章节 | 第 X 章 |
| 状态一致性 | 一致/有变更 |

### 待办事项（从上次接力）
- [ ] <待办 1>
- [ ] <待办 2>

### 建议下一步
- <下步计划>
```

#### Step 4: 自动提示待办
如果有未完成的待办事项，按优先级提示用户。

### status 模式

输出接力文件的摘要信息，不做任何修改：
```markdown
## 接力状态

| 项目 | 值 |
|------|-----|
| 接力文件 | 存在/不存在 |
| 上次保存 | <时间> |
| 对应章节 | 第 X 章 |
| 待办事项 | <N> 项 |
| 状态一致性 | 一致/可能过期 |
```

## 安全边界
- save 不覆盖正文文件，只写 handoff.md
- load 不自动修改任何文件，只读取和报告
- 检测到外部修改时提醒用户，不静默覆盖
- 不伪造会话 ID 或时间戳

## 无 MCP 降级
如果 codeagent-wrapper 不可用：
- 本命令为纯 Claude 单模型操作，无需外部模型
- 所有保存、加载、校验均在本地完成
- 无降级影响
