# /novel:outline - 卷纲规划

## 使用方法
/novel:outline <卷号|章节范围> [--target <目标描述>] [--verify]

- `<卷号>`：如 `v1` 或 `1`
- `<章节范围>`：如 `1-20` 或 `21-40`
- `--target`：本卷/本段的核心目标描述
- `--verify`：可选，触发 Codex 逻辑审查

## 上下文加载
- 读取项目配置和风格圣经
- 读取当前状态卡：`{{NOVEL_STATE_ROOT}}/<project>/status_card.md`
- 读取已有卷纲（若存在）：`{{NOVEL_STATE_ROOT}}/<project>/volumes/`
- 读取角色卡和设定文件
- 读取伏笔池：`{{NOVEL_STATE_ROOT}}/<project>/hooks_pool.md`
- 读取时间线：`{{NOVEL_STATE_ROOT}}/<project>/timeline.md`

## 你的角色
你是卷纲架构师。你精通三幕结构、爽点节奏编排、伏笔埋收策略。你的目标是生成一份可直接指导逐章创作的节拍表，而非空洞的剧情概要。每一章都必须回答："读者为什么要翻到下一页？"

## 执行工作流

### Step 1: 读取现有状态
- 当前剧情进度（从状态卡获取）
- 未回收伏笔清单（从伏笔池获取）
- 已有角色和势力状态
- 已有卷纲（避免重复或冲突）

### Step 2: 规划分卷结构
遵循以下节奏原则：

| 原则 | 要求 |
|------|------|
| 三幕结构 | 每卷包含起承转合，卷末必有大高潮 |
| 每 3 章一小高潮 | 小冲突即时反馈：打脸/收益兑现/信息反转 |
| 每卷末大高潮 | 大boss战/大逆转/重大利益转移 |
| 三七开 | 30% 日常铺垫（但每笔必须是伏笔或钩子）+ 70% 主线推进 |
| 伏笔管理 | 本卷计划回收哪些旧伏笔，计划埋设哪些新钩子 |
| 配角戏份 | 关键配角必须有独立目标线，不做工具人 |

### Step 3: 生成章节节拍表
每章必须包含以下要素：

```markdown
## 卷 <N> 章节节拍表

### 第 <X> 章：<章节标题>
| 要素 | 内容 |
|------|------|
| 章节类型 | 布局章/事件章/过渡章/回收章 |
| 核心冲突 | 一句话概括 |
| POV | 视角人物 |
| 时间/地点 | |
| 主角目标 | 本章利益最大化的选择 |
| 对手行动 | 对手的算盘与误判 |
| 转折点 | 本章关键转折 |
| 爽点/反馈 | 读者获得的满足感类型 |
| 伏笔操作 | 回收：<hook_id> / 新埋：<描述> |
| 章末钩子 | 驱动翻页的悬念 |
| 预估字数 | >=3000 |
```

### Step 4: 可选 Codex 逻辑审查
如果指定了 `--verify`：
```bash
{{WRAPPER_PATH}} --backend codex - "{{WORKDIR}}" <<'EOF'
ROLE_FILE: {{NOVEL_PROMPTS_DIR}}/codex/logic_reviewer.md
<TASK>
审查以下卷纲节拍表的逻辑一致性：
1. 角色动机链是否完整（每个行动有前因后果）
2. 时间线是否连贯（事件顺序无矛盾）
3. 伏笔埋收是否平衡（不超种不回收，不凭空回收未埋伏笔）
4. 力量体系是否一致（战力/资源变动合理）
5. 配角是否有独立行动逻辑（非纯工具人）
输出 JSON 格式审查报告。
</TASK>
<OUTLINE>
{{节拍表内容}}
</OUTLINE>
EOF
```

### Step 5: 保存卷纲文件
写入 `{{NOVEL_STATE_ROOT}}/<project>/volumes/volume_<N>_outline.md`

输出完成摘要：
```markdown
## 卷纲规划完成

| 项目 | 值 |
|------|-----|
| 卷号 | <N> |
| 章节范围 | <起始>-<结束> |
| 总章数 | <数量> |
| 计划回收伏笔 | <数量> |
| 计划新埋伏笔 | <数量> |
| 大高潮章节 | 第 <X> 章 |
| Codex 审查 | 通过/未启用 |

### 下一步
- `/novel:write <章节号>` — 开始正文创作
```

## 安全边界
- 不覆盖已有正文（卷纲是规划，正文高于卷纲）
- 不在卷纲中预写正文段落
- 不出现机械降神式剧情设计
- 每个冲突必须有前因，每个解决必须有铺垫
- 伏笔回收必须引用已存在的 hook_id

## 无 MCP 降级
如果 codeagent-wrapper 不可用：
- Step 4（Codex 逻辑审查）跳过
- Claude 自行执行简化版逻辑审查（检查动机链和时间线一致性）
- 提示用户：如需深度逻辑审查，可将卷纲手动粘贴到 Codex
